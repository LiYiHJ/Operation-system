# 上线前最后操作顺序（建议）

1. **环境冻结与依赖核对（T-3 天）**
   - 固定镜像/依赖版本；
   - 执行启动自检脚本（前端/后端/DB）。

2. **主库基线回放（T-2 天）**
   - 回放 1 份 demo + 2 份真实报表；
   - 校验 6 张事实表 + 扩展事实表非零。

3. **真实外部联通演练（T-2 天）**
   - Ozon 最小 scope 真凭证验收；
   - 销售后台真实 URL 推送验收；
   - 验证 traceId/idempotencyKey/失败日志。

4. **全站 UAT 回归（T-1 天）**
   - 页面顺序：Dashboard → ABC → Funnel → Price → Ads → Reminder → Strategy → Decision → Settings；
   - 每页至少 1 条非空业务路径。

5. **闭环压测与失败恢复演练（T-1 天）**
   - 批量生成/确认/推送（小批次）；
   - 人工触发部分失败并执行重试。

6. **门禁评审（上线日）**
   - 对照《final_production_gate_table.md》逐项签字；
   - P0 必须清零；P1 需有明确缓解与值班方案。

7. **灰度放量与观察（上线后 24h）**
   - 先小流量、再逐步放量；
   - 实时观察导入成功率、push 失败率、异常告警。
