    Refactor import mapping summary, add dynamicCompanion/excludeFromSemanticGate flags, and frontend/type updates
    
    ### Motivation
    - Unify and centralize field-mapping summary logic on the backend to provide richer bundle metadata and avoid duplicated calculations.
    - Support fields that should be treated as dynamic companions or excluded from semantic gating so they don't affect semantic metrics.
    - Surface new mapping metadata to the frontend and typings so the UI can correctly ignore/label companion/excluded columns.
    - Remove noisy debug logs and add repository ignore rules for local dev artifacts.
    
    ### Description
    - Added `dynamicCompanion` and `excludeFromSemanticGate` flags to the frontend `normalizeFieldMappings` and to the `FieldMapping` type in `types/index.ts`, and updated `isIgnoredField` logic to account for these flags.
    - Removed debug `console.log` calls from the upload and confirm flows in `DataImportV2.tsx`.
    - Introduced `_build_field_mapping_summary` in `ImportService` and refactored `_build_bundle` to use it, consolidating computation of `mappedTargets`, `mappedCount`, `unmappedCount`, `mappedCanonicalFields`, `topUnmappedHeaders`, `semanticStatus`, `semanticMetrics`, and related fields.
    - Updated `parse_import_file` and the manual-override handling path to consume and propagate the new bundle/summary fields (including `mappedCanonicalFields`, `topUnmappedHeaders`, `semanticMetrics`, `semanticStatus`, `semanticGateReasons`, and `riskOverrideReasons`).
    - Minor defensive fixes in summary calculations (e.g. safe iteration over possibly-None lists and confidence averaging).
    - Added `.gitignore` entries for virtual environments, local uploads, DB files, and local docs diffs.
    
    ### Testing
    - Ran backend unit tests for import-related logic with `pytest` and they passed.
    - Ran frontend type checks and build (`tsc` / `npm run build`) to validate type additions and removed logs and they passed.
    - Performed import flow smoke test against staging import API to verify mapping summary propagation and UI handling of companion/excluded fields and it succeeded.
