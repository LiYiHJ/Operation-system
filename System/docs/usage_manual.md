# 运营系统详细使用说明报告

> 适用版本：当前仓库主干（前后端分离架构）
> 
> 项目路径：`C:\Operation-system\System`

---

## 1. 文档目标

本报告用于指导运营、开发、测试在**真实库场景**中使用系统，覆盖以下核心流程：

1. 系统启动与登录
2. 系统设置中的数据接入中心
3. 文件导入（兜底）
4. Ozon API 同步（主路径）
5. 利润定价模块使用
6. 策略确认与销售后台推送
7. 常见问题定位与验收建议

---

## 2. 系统整体结构

系统采用前后端分离，核心模块如下：

- 前端：React + Ant Design（运营页面、系统设置、利润定价、策略执行）
- 后端：Flask API（分析、导入、提醒、策略、集成）
- 数据库：PostgreSQL（推荐真实库 `ecom_v51_prod`）
- 集成：
  - Ozon 数据接入（按业务域 scope 同步）
  - 销售后台推送（执行动作回写）

## 3. 启动与登录

### 3.1 后端启动（真实库）

在 PowerShell 执行：

```powershell
cd C:\Operation-system\System
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "$PWD\src"
$env:DATABASE_URL = "postgresql+psycopg://ecom_user:strong_password@127.0.0.1:5432/ecom_v51_prod"
```

启动后应看到：

- API 地址：`http://localhost:5000/api`

```powershell
cd C:\Operation-system\System\frontend
```

浏览器打开：`http://localhost:5173`

## 4. 系统设置：数据接入中心（主入口）

路径：`系统设置 -> 数据接入中心`

该页面用于统一管理 API 与文件导入，不再建议把“手工导入”作为主流程。

### 4.1 平台连接与权限

需配置：

- Provider（Ozon）
- 是否启用数据源
- 自动拉取开关
- 同步频率（手动/每小时/每日）
- Client ID
- Seller ID
- 只读采集 Token（read token）
- 动作执行 Token（action token）
- 销售后台推送 URL
- 是否使用 Mock Ozon（仅联调用）

操作：

1. 填写上述字段并点击“保存配置”
2. 点击“权限校验”检查 read/action token 可用性

### 4.2 同步范围（按业务域）

支持按业务域勾选同步：

1. 商品与上品中台
2. 订单与履约中台
3. 促销与价格中台
4. 服务与售后中台
5. 报表与经营分析中台

操作：

1. 勾选一个或多个业务域
2. 点击“立即同步一次”
3. 在“最近同步记录”查看状态、导入行数、消息

### 4.3 同步日志与导入日志

- 最近同步记录：用于看 API 拉取执行状态
- 最近导入记录：用于看导入批次状态

建议重点关注：

- 状态（success/failed）
- message（scope、导入行数、报错摘要）

## 5. 文件导入（兜底流程）

路径：`系统设置 -> 文件导入`

适用场景：

- API 临时不可用
- 需要补录历史文件
- 对账/回补

推荐文件目录：`C:\Operation-system\System\data`

完整流程：

1. 选择文件（xlsx/csv/json）
2. 点击开始解析
3. 进入字段识别/映射
4. 确认导入
5. 查看导入结果与日志

验证建议：

- 浏览器 Network 确认上传请求真实发出
- 导入后查询 DB 批次与业务表是否增量
- 至少一个页面能读取到导入结果

---

## 6. Ozon API 同步（主流程）

### 6.1 推荐流程

1. 系统设置中完成 Ozon 凭证配置
2. 权限校验通过
3. 勾选业务域 scope
4. 立即同步一次
5. 查看同步日志
6. 进入业务页面验证结果

### 6.2 成功判定

满足以下条件可判定同步有效：

- 同步状态 `success`
- 同步日志 message 中有 scope 与数据量
- DB 有新增批次/业务数据
- 页面或接口可展示最新数据

### 6.3 失败排查

按优先级检查：

1. 凭证是否过期（Client ID / Token）
2. scope 是否超出当前账号权限
3. 目标 API 路径是否匹配当前 Ozon 版本
4. 返回字段是否与映射逻辑一致
5. 入库约束是否触发冲突

## 7. 利润定价模块使用

路径：`利润定价工作台`

页面分为两个顶层视图：

1. 定价决策
2. 成本与规则

### 7.1 定价决策

核心目标：快速完成“当前价格是否可执行”的判断。

主要区域：

- 平台价格结构（原价/活动价/券后价/成交价）
- KPI（净利润、净利率、建议售价、保本价、保本折扣）
- 主求解输入（模式、目标值、sale_price、list_price）
- 多情景对比
- 折扣模拟
- 调价敏感度分析
- 动作闭环按钮（保存、推送、写策略）

### 7.2 成本与规则

用于深度调整与复盘：

- 上品前/上品后场景切换
- 运输成本引擎（按方案自动回填）
- 变量费率明细 + 固定成本明细
- 参数来源标签（平台自动值/店铺默认值/SKU覆盖值/本次模拟值）
- 风险扫描

---

## 8. 运输成本引擎说明

输入维度：

- 仓库
- 渠道商
- 服务等级
- 到门/到点
- 产品分层
- 重量
- 体积
- 敏感货
- 货值

输出回填：

- 头程成本
- 履约成本
- 仓储成本
- 最后一公里成本
- 操作费等

这些值会进入固定成本总和并影响建议售价。

## 11. 常见问题

### Q1：同步显示成功，但页面没数据

优先检查：

- 同步 scope 是否包含该页面依赖的数据域
- 数据是否写入了页面查询依赖表
- 页面筛选条件是否过滤掉了新数据

### Q2：推送失败

优先检查：

- 销售后台 URL 是否可达
- action token 是否可用
- 请求体字段是否满足对方协议

### Q3：导入提示成功但结果不一致

优先检查：

- 解析阶段映射是否正确
- confirm 后批次统计（success/error）
- 约束冲突导致的更新/跳过

---

## 12. 结论

当前系统建议采用“API 主导 + 文件兜底”的运营模式：

- 日常靠 Ozon API 定时/手动同步
- 异常或补录走文件导入
- 利润页使用自动回填 + 场景试算完成决策
- 决策动作通过推送闭环落地并可审计
