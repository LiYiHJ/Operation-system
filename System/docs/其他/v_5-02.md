# V5.1 跨境电商智能运营系统：系统总设计与实施蓝图

## 1. 项目定位

V5.1 的目标不是做一个“报表分析器”，而是构建一个 **可持续运行、可部署、可扩展、以 SKU 为核心的跨境电商智能运营系统**。

系统解决四类问题：

1. **数据接入难**：Ozon 导出的 Excel/CSV/JSON 俄语字段、表头不固定、货币/百分比/换行导致导入失败。
2. **分析维度窄**：过去版本更偏单文件、单次分析，难以支撑店铺/类目/SKU/广告/库存/订单联动管理。
3. **利润口径不完整**：已有价格与利润设计，但需要升级为“真净利润 + 保本价 + 目标净利润反推售价”的闭环系统。
4. **策略不闭环**：系统不能只给建议，还要输出任务、优先级、观察指标、复盘记录。

## 5. 数据接入设计

### 5.1 支持的数据输入

- Ozon API 抓取 JSON（接入现有 latest.json / raw / ads 目录）
- Ozon 后台“我的分析”Excel/CSV
- 广告报表
- 库存报表
- 订单报表
- 财务报表
- 价格主档 / 成本主档 / 竞品价格 / 促销计划
- ZIP 批量导入

### 5.2 导入总流程

```text
上传文件
→ 文件识别
→ 平台识别
→ 表头检测
→ 多行表头合并
→ 字段映射
→ 单元格清洗
→ 主键识别
→ 类型转换
→ 核心字段校验
→ 写入原始表和标准表
→ 输出导入结果与失败诊断
```

### 5.3 导入诊断必须输出

每次导入都生成一份“诊断报告”：

- 文件名 / 工作表名
- 检测到的表头行
- 检测到的平台
- 成功映射字段数
- 未识别字段列表
- 主键字段是否存在
- 数据行错误统计
- 失败原因与修正建议

### 5.4 关键导入规则

- 表头扫描：前 20 行
- 必须支持多级表头拼接
- 必须优先用 SKU / Артикул / seller_sku / offer_id 做主键
- 禁止使用商品名称作为主键
- 金额类统一清洗 `₽`, `руб`, 空格, 换行
- 百分比统一转换为小数存储，同时保留展示格式
- 日期支持俄语文本日期解析
- 所有导入记录必须带 batch_id

## 8. 利润中心设计

## 8.1 核心原则

利润不能只保留一个口径，必须分层：

1. **基础利润**：售价 - 基础成本 - 平台佣金
2. **贡献利润**：基础利润 - 广告分摊 - 促销补贴
3. **履约后利润**：贡献利润 - 仓储/履约/物流/包材
4. **真净利润**：履约后利润 - 退货损失 - 取消损失 - 税费 - 汇损 - 其他分摊

### 8.2 变量与固定成本拆分

为了可求解，所有费用统一拆成两类：

- **变量费率 V**：与售价按比例变化的费用
  - 平台佣金率
  - 支付/服务费率
  - 税费率
  - 广告费率（按销售额分摊时）
  - 汇损费率

- **固定成本 F**：与单件售价无关的单件成本
  - 采购成本
  - 头程
  - 包材
  - 履约费
  - 仓储费
  - 尾程物流
  - 单件广告分摊
  - 退货预期损失
  - 取消预期损失
  - 促销固定补贴
  - 其他单件成本

### 8.3 真净利润核心公式

```text
真净利润 = 售价 × (1 - V) - F
```

其中：

- `V` = 所有比例型费用之和
- `F` = 所有单件固定成本之和

### 8.4 四个关键求解

#### A. 已知售价，求真净利润

```text
NP = P × (1 - V) - F
```

输出：

- 单件真净利润
- 净利率
- 是否亏损

#### B. 保本价

```text
P_break_even = F / (1 - V)
```

输出：

- 保本售价
- 当前售价高于/低于保本价多少

#### C. 已知目标单件净利润，反推售价

```text
P_target_profit = (T + F) / (1 - V)
```

#### D. 已知目标净利率，反推售价

```text
P_target_margin = F / (1 - V - M)
```

若 `M >= 1 - V`，则目标不现实，系统直接提示无解。

### 8.5 折扣求解器

设原价为 `L`，折扣系数为 `q`，成交价 `P = L × q`。

#### 保本最低折扣

```text
q_break_even = F / (L × (1 - V))
```

#### 达到目标净利润的最低折扣

```text
q_target_profit = (T + F) / (L × (1 - V))
```

系统要直接翻译成业务语言：

- 最低只能打几折不亏
- 若要达到目标净利润，最低只能打几折
- 当前促销力度是否危险

### 8.6 数据聚合与回退逻辑

利润计算前必须统一口径，并支持样本不足回退：

1. SKU 近30天真实数据
2. SKU 近90天
3. SPU 均值
4. 类目均值
5. 店铺默认规则

### 8.7 广告与退货分摊规则

广告支持两种口径：

- 按单件分摊
- 按销售额费率分摊

退货/取消采用期望损失：

```text
退货预期损失 = 退货率 × 单次退货平均损失
取消预期损失 = 取消率 × 单次取消平均损失
```

- 订单数

- 下单率
- 广告 ROI

- 评分
- 真净利润

### 9.3 漏斗诊断区

识别：

- 曝光不足

同时展示：

- 当前真净利润

- 当前是否亏损

### 9.5 广告与流量区

### 9.6 库存与供给区

- 当前库存 / FBO / FBS

- 安全库存

### 9.7 风险控制区

- 高退货风险
- 高取消风险
- 低评分风险

### 9.8 策略组合区

输出的不是单条建议，而是一组策略：

- 主策略

## 10. 分析引擎设计

### 10.1 漏斗分析引擎

漏斗：

```text
展示 → 商品卡访问 → 加购 → 下单 → 成交
```

输出：

- 漏斗各环节转化率
- 最大断点
- 对应修复建议

### 10.2 ABC / XYZ / 增长矩阵引擎

- ABC：按金额 / 利润 / 销量
- XYZ：按波动稳定性
- 增长矩阵：增长率 × 销售额

用于区分：

- 核心爆款
- 稳定利润款
- 潜力款
- 尾部淘汰款

### 10.3 定价策略引擎

不是只看价格指数，还要联动：

- CTR
- 加购率
- 下单率
- 净利润
- 库存天数
- 广告依赖度
- 评分与退货率

输出：

- 是否适合提价
- 是否适合降价
- 是否应该维持价格
- 是否应该先修内容/广告/库存
- 建议幅度与观察周期

### 10.4 广告效率引擎

- ROI / ROAS
- 广告收入占比
- 广告依赖度
- 高花费低产出 SKU
- 可提投 SKU
- 应停投 SKU

### 10.5 库存引擎

- 缺货风险
- 滞销风险
- 建议补货量
- 库存周转
- 清仓候选池

### 10.6 店铺健康引擎

综合：

- 销售趋势
- 评分
- 退货率
- 缺货率
- 广告效率
- 净利润
- 爆款依赖度

### 25.1 主关系

```text
dim_platform 1 ── N dim_shop

dim_shop 1 ── N dim_product

dim_product 1 ── N dim_sku

dim_category 1 ── N dim_product

dim_shop 1 ── N fact_sku_daily

dim_sku 1 ── N competitor_price_snapshot

dim_sku 1 ── N promotion_event

import_batch 1 ── N import_batch_file
import_batch_file 1 ── N import_error_log

dim_shop / dim_sku / dim_category ── strategy_task
```

### 25.2 主键策略

- 所有事实表与主表统一采用内部 `id` 作为数据库主键
- 业务唯一识别优先用：
  1. `sku`
  2. `offer_id`
  3. `seller_sku`
  4. `external_sku`
- 任何情况下不允许把商品标题作为唯一键

### 25.3 建议索引

#### 高频索引

- `dim_sku(shop_id, sku)`
- `dim_sku(shop_id, offer_id)`
- `fact_sku_daily(date_id, shop_id, sku_id)`
- `fact_profit_snapshot(date_id, shop_id, sku_id)`
- `strategy_task(shop_id, priority, status)`
- `import_batch(status, started_at)`

#### 查询与筛选索引

- `fact_ads_daily(shop_id, sku_id, date_id)`
- `fact_inventory_daily(shop_id, sku_id, date_id)`
- `fact_orders_daily(shop_id, sku_id, date_id)`
- `sku_price_master(shop_id, sku_id)`

### 25.4 数据版本与覆盖策略

- 所有导入必须写入 `batch_id`
- 原始文件信息保留在 `import_batch_file`
- 同一天同 SKU 的事实数据采用“按来源 + 批次版本”管理
- 提供两种模式：
  - `append`：保留历史版本
  - `upsert`：按 `date_id + shop_id + sku_id + source_type` 覆盖当前有效版本

### 25.5 汇率与币种策略

- 数据库存储至少保留两种金额：
  - 原币金额（RUB）
  - 统一展示币种金额（CNY）
- 汇率来源需要独立表 `fx_rate_daily`

#### `fx_rate_daily`

- `id`

## 26. 导入中心详细设计

导入中心是 V5.1 第一优先级模块，因为你过去版本最大的问题就是“导不进来”或“导进来但字段乱了”。

### 26.1 页面流程

#### 页面一：上传页

用户选择：

- 文件上传
- 文件夹批量导入
- ZIP 批量导入
- 从现有 Ozon JSON 目录导入

上传后立即进入预处理。

#### 页面二：预览页

系统展示：

- 文件基本信息
- 检测到的平台
- 工作表列表
- 每个工作表候选表头行
- 前 20 行预览
- 检测到的关键字段（SKU/订单金额/展示等）

#### 页面三：映射确认页

用户可修正：

- 真正的表头行
- 主键字段
- 未识别字段的映射
- 金额/百分比/日期列的类型

#### 页面四：导入结果页

显示：

- 成功导入多少行
- 跳过多少行
- 哪些列未识别
- 哪些值清洗失败
- 哪些核心字段缺失
- 可下载错误日志

### 26.2 文件识别器设计

#### 输入支持

- `.xlsx`
- `.xlsm`
- `.csv`
- `.tsv`
- `.json`
- `.zip`
- 指定 JSON 目录

#### 输出结构

```json
{
  "file_type": "xlsx",
  "platform_guess": "ozon",
  "sheet_count": 3,
  "candidate_sheets": ["分析", "库存", "广告"]
}
```

### 26.3 平台识别器设计

平台识别依赖：

- 文件名关键词
- 表头关键词
- JSON 顶层结构
- 字段语言特征

#### Ozon 识别信号示例

- 文件名中包含 `ozon`, `analytics`, `my analysis`
- 表头中包含俄语关键词：
  - `Показы`
  - `Артикул`
  - `Заказано`
  - `Добавлено в корзину`
- JSON 中包含 `seller_info`, `analytics_data_sku`, `campaigns_stats` 等键名

### 26.4 表头定位器设计

#### 目标

解决：

- 表头不在第一行
- 合并单元格
- 上方有大标题/说明文字
- 表头跨两行或三行

#### 规则

1. 扫描前 20 行
2. 计算每行“字段命中密度”
3. 选择分值最高的 1~3 行作为候选表头
4. 若两行连续高分，则尝试多级表头合并
5. 自动拼接多级表头，如：
   - `搜索结果和目录` + `展示次数` → `搜索结果和目录_展示次数`

#### 输出结构

```json
{
  "candidate_header_rows": [4, 5],
  "final_header_mode": "multi_row_merge",
  "confidence": 0.91
}
```

### 26.5 字段映射器设计

字段映射采用三层：

```text
原始字段名 → 规范中文字段 → 系统内部字段
```

#### 例子

- `Артикул` → `SKU编码` → `sku`
- `Заказано на сумму` → `订购金额` → `ordered_revenue`
- `Показы` → `总展示次数` → `impressions_total`
- `Добавлено в корзину` → `加购数` → `add_to_cart_total`

#### 映射优先级

1. 精确字典命中
2. 去空格/去换行后命中
3. 俄语归一化后命中
4. 模糊匹配（rapidfuzz）
5. 上下文语义判断（同组列）
6. 人工确认

### 26.6 单元格清洗器设计

#### 金额清洗

处理：

- `1 234 ₽`
- `₽ 3 560`
- `3 210,55`
- `1
  234`

统一为数值型。

#### 百分比清洗

处理：

- `12,4 %`
- `5%`
- `0.34`

统一存储为小数，如 `0.124`。

#### 日期清洗

支持：

- 标准日期
- 俄语文本日期
- Excel serial date

### 26.7 主键识别器设计

#### 规则

优先级：

1. `sku`
2. `Артикул`
3. `offer_id`
4. `seller_sku`
5. `SKU编码`

#### 失败提示

如果没有识别到主键，导入不应直接崩溃，而应提示：

- 未发现 SKU 主键字段
- 可手动指定以下候选列
- 若无主键，将无法进入商品中心

### 26.8 导入校验器设计

#### 核心字段校验按数据类型不同而不同

##### 商品分析表最小必需字段

- SKU
- 订购金额/销售额（至少一种）

##### 漏斗分析增强字段

- 展示次数
- 商品卡访问 / 点击
- 加购数
- 订单数

##### 定价利润增强字段

- 当前售价
- 成本或价格主档

##### 库存增强字段

- 当前库存
- 可售天数或库存量

### 26.9 导入失败分类

系统把失败原因分成：

- `NO_HEADER_FOUND`
- `NO_PRIMARY_KEY`
- `NO_REQUIRED_METRICS`
- `INVALID_NUMBER_FORMAT`
- `INVALID_DATE_FORMAT`
- `EMPTY_SHEET`
- `UNKNOWN_PLATFORM`
- `MAPPING_CONFLICT`

页面上必须给出可读解释。

### 26.10 导入后写库顺序

```text
原始文件存档
→ import_batch / import_batch_file
→ 字段映射结果
→ 主数据 upsert（shop/product/sku/category）
→ 事实表写入
→ 指标计算任务入队
→ 导入完成报告
```

## 27. 利润中心详细计算设计

### 27.1 输入字段分组

#### 价格组

- 当前售价 `sale_price`
- 原价 `list_price`
- 成交均价 `avg_sold_price`
- 促销折扣 `discount_ratio`

#### 固定成本组 F

- 采购成本 `purchase_cost`
- 头程 `first_leg_cost`
- 包材 `packaging_cost`
- 履约 `fulfillment_cost`
- 仓储 `warehouse_cost`
- 尾程物流 `last_mile_cost`
- 固定广告分摊 `ad_cost_fixed`
- 退货预期损失 `return_loss_expected`
- 取消预期损失 `cancel_loss_expected`
- 促销固定补贴 `promo_subsidy_fixed`
- 其他固定成本 `other_fixed_cost`

#### 变量费率组 V

- 平台佣金率 `platform_commission_rate`
- 支付服务费率 `service_fee_rate`
- 税费率 `tax_rate`
- 按销售额广告费率 `ad_rate`
- 汇损费率 `fx_loss_rate`
- 其他比例成本 `other_variable_rate`

### 27.2 口径计算顺序

#### A. 基础利润

```text
基础利润 = 售价 - 采购成本 - 平台佣金 - 基础固定费
```

#### B. 贡献利润

```text
贡献利润 = 基础利润 - 广告分摊 - 促销补贴
```

#### C. 履约后利润

```text
履约后利润 = 贡献利润 - 履约 - 仓储 - 物流 - 包材
```

#### D. 真净利润

统一求解时采用：

```text
真净利润 = 售价 × (1 - V) - F
```

### 27.3 求解器输入模式

#### 模式 1：输入目标单件净利润

输出建议售价：

```text
建议售价 = (目标净利润 + F) / (1 - V)
```

#### 模式 2：输入目标净利率

输出建议售价：

```text
建议售价 = F / (1 - V - 目标净利率)
```

#### 模式 3：输入目标 ROI

输出建议售价：

```text
建议售价 = F × (1 + 目标ROI) / (1 - V)
```

### 27.4 折扣模拟器

设原价 `L`，折扣系数 `q`。

```text
成交价 = L × q
真净利润 = L × q × (1 - V) - F
```

输出：

- 成交价
- 真净利润
- 净利率
- 是否亏损
- 保本剩余空间

### 27.5 业务翻译层

利润中心页面不只显示公式结果，还要翻译成可执行信息：

- 最低只能打几折不亏
- 若要达到目标净利润，最低只能打几折
- 当前价格低于建议售价多少
- 若维持当前价格，主要是广告在吃利润还是履约在吃利润
- 该商品更适合提价、维持、清仓还是先修内容
