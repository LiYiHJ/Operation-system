# 项目打包交接清单

## 一、文档目的

本文档用于指导你把当前系统整理成一个适合交给 Codex 接手的 handoff 包。

目标不是“把整个电脑打包”，而是：

- 把必要代码和说明交完整
- 把敏感信息剥离干净
- 让 Codex 能快速理解项目
- 让它优先按正确方向接线和实现
- 降低“误删功能 / 误重写 / 误判结构”的风险

---

## 二、打包前总原则

1. 不要只发一句执行指令给 Codex  
2. 必须一起发：代码 + 文档 + 启动说明 + 测试说明  
3. 不要包含真实密钥  
4. 不要包含缓存和构建垃圾  
5. 不要省略示例导入文件  
6. 不要省略当前功能清单和设计要求  
7. 不要默认 Codex 会自己理解你的业务重点，必须把重点写在 handoff 文档里

---

## 三、交接包建议目录结构

建议在项目根目录创建一个 `handoff/` 目录，并把交接文档统一放进去。

推荐结构：

```text
project-root/
├─ frontend/
├─ backend/
├─ sample_data/
├─ handoff/
│  ├─ 00_正式设计要求_V2.md
│  ├─ 01_完整功能清单.md
│  ├─ 02_项目打包交接清单.md
│  ├─ 04_启动说明.md
│  ├─ 05_导入链路测试说明.md
│  └─ 06_已知问题与优先级.md（可选）
├─ .env.example
├─ README.md
└─ 其他必要脚本
