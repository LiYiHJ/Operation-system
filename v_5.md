# V5.1 跨境电商智能运营系统：系统总设计与实施蓝图

## 1. 项目定位

V5.1 的目标不是做一个“报表分析器”，而是构建一个 **可持续运行、可部署、可扩展、以 SKU 为核心的跨境电商智能运营系统**。

系统解决四类问题：

1. **数据接入难**：Ozon 导出的 Excel/CSV/JSON 俄语字段、表头不固定、货币/百分比/换行导致导入失败。
2. **分析维度窄**：过去版本更偏单文件、单次分析，难以支撑店铺/类目/SKU/广告/库存/订单联动管理。
3. **利润口径不完整**：已有价格与利润设计，但需要升级为“真净利润 + 保本价 + 目标净利润反推售价”的闭环系统。
4. **策略不闭环**：系统不能只给建议，还要输出任务、优先级、观察指标、复盘记录。

## 5. 数据接入设计

### 5.1 支持的数据输入

- Ozon API 抓取 JSON（接入现有 latest.json / raw / ads 目录）
- Ozon 后台“我的分析”Excel/CSV
- 广告报表
- 库存报表
- 订单报表
- 财务报表
- 价格主档 / 成本主档 / 竞品价格 / 促销计划
- ZIP 批量导入

### 5.2 导入总流程

```text
上传文件
→ 文件识别
→ 平台识别
→ 表头检测
→ 多行表头合并
→ 字段映射
→ 单元格清洗
→ 主键识别
→ 类型转换
→ 核心字段校验
→ 写入原始表和标准表
→ 输出导入结果与失败诊断
```

### 5.3 导入诊断必须输出

每次导入都生成一份“诊断报告”：

- 文件名 / 工作表名
- 检测到的表头行
- 检测到的平台
- 成功映射字段数
- 未识别字段列表
- 主键字段是否存在
- 数据行错误统计
- 失败原因与修正建议

### 5.4 关键导入规则

- 表头扫描：前 20 行
- 必须支持多级表头拼接
- 必须优先用 SKU / Артикул / seller_sku / offer_id 做主键
- 禁止使用商品名称作为主键
- 金额类统一清洗 `₽`, `руб`, 空格, 换行
- 百分比统一转换为小数存储，同时保留展示格式
- 日期支持俄语文本日期解析
- 所有导入记录必须带 batch_id

## 8. 利润中心设计

## 8.1 核心原则

利润不能只保留一个口径，必须分层：

1. **基础利润**：售价 - 基础成本 - 平台佣金
2. **贡献利润**：基础利润 - 广告分摊 - 促销补贴
3. **履约后利润**：贡献利润 - 仓储/履约/物流/包材
4. **真净利润**：履约后利润 - 退货损失 - 取消损失 - 税费 - 汇损 - 其他分摊

### 8.2 变量与固定成本拆分

为了可求解，所有费用统一拆成两类：

- **变量费率 V**：与售价按比例变化的费用
  - 平台佣金率
  - 支付/服务费率
  - 税费率
  - 广告费率（按销售额分摊时）
  - 汇损费率

- **固定成本 F**：与单件售价无关的单件成本
  - 采购成本
  - 头程
  - 包材
  - 履约费
  - 仓储费
  - 尾程物流
  - 单件广告分摊
  - 退货预期损失
  - 取消预期损失
  - 促销固定补贴
  - 其他单件成本

### 8.3 真净利润核心公式

```text
真净利润 = 售价 × (1 - V) - F
```

其中：
- `V` = 所有比例型费用之和
- `F` = 所有单件固定成本之和

### 8.4 四个关键求解

#### A. 已知售价，求真净利润

```text
NP = P × (1 - V) - F
```

输出：
- 单件真净利润
- 净利率
- 是否亏损

#### B. 保本价

```text
P_break_even = F / (1 - V)
```

输出：
- 保本售价
- 当前售价高于/低于保本价多少

#### C. 已知目标单件净利润，反推售价

```text
P_target_profit = (T + F) / (1 - V)
```

#### D. 已知目标净利率，反推售价

```text
P_target_margin = F / (1 - V - M)
```

若 `M >= 1 - V`，则目标不现实，系统直接提示无解。

### 8.5 折扣求解器

设原价为 `L`，折扣系数为 `q`，成交价 `P = L × q`。

#### 保本最低折扣

```text
q_break_even = F / (L × (1 - V))
```

#### 达到目标净利润的最低折扣

```text
q_target_profit = (T + F) / (L × (1 - V))
```

系统要直接翻译成业务语言：
- 最低只能打几折不亏
- 若要达到目标净利润，最低只能打几折
- 当前促销力度是否危险

### 8.6 数据聚合与回退逻辑

利润计算前必须统一口径，并支持样本不足回退：

1. SKU 近30天真实数据
2. SKU 近90天
3. SPU 均值
4. 类目均值
5. 店铺默认规则

### 8.7 广告与退货分摊规则

广告支持两种口径：
- 按单件分摊
- 按销售额费率分摊

退货/取消采用期望损失：

```text
退货预期损失 = 退货率 × 单次退货平均损失
取消预期损失 = 取消率 × 单次取消平均损失
```

识别：
- 曝光不足

同时展示：
- 当前真净利润

### 9.7 风险控制区

- 高退货风险
- 高取消风险
- 低评分风险

## 10. 分析引擎设计

### 10.1 漏斗分析引擎

漏斗：

```text
展示 → 商品卡访问 → 加购 → 下单 → 成交
```

输出：
- 漏斗各环节转化率
- 最大断点
- 对应修复建议

### 10.2 ABC / XYZ / 增长矩阵引擎

- ABC：按金额 / 利润 / 销量
- XYZ：按波动稳定性
- 增长矩阵：增长率 × 销售额

用于区分：
- 核心爆款
- 稳定利润款
- 潜力款
- 尾部淘汰款

### 10.3 定价策略引擎

不是只看价格指数，还要联动：
- CTR
- 加购率
- 下单率
- 净利润
- 库存天数
- 广告依赖度
- 评分与退货率

输出：
- 是否适合提价
- 是否适合降价
- 是否应该维持价格
- 是否应该先修内容/广告/库存
- 建议幅度与观察周期

### 10.4 广告效率引擎

- ROI / ROAS
- 广告收入占比
- 广告依赖度
- 高花费低产出 SKU
- 可提投 SKU
- 应停投 SKU

### 10.5 库存引擎

- 缺货风险
- 滞销风险
- 建议补货量
- 库存周转
- 清仓候选池

### 10.6 店铺健康引擎

综合：
- 销售趋势
- 评分
- 退货率
- 缺货率
- 广告效率
- 净利润
- 爆款依赖度

## 13. 真实使用顺序

### 场景 A：日常运营

1. 看首页异常告警
2. 进入策略中心看 P0/P1
3. 进入商品作战室看重点 SKU
4. 在利润中心检查是否亏损 / 是否可以调价
5. 建立调价或补货任务
6. 3天后复盘效果

### 场景 B：做促销前

1. 进入利润中心批量选择商品
2. 模拟 95折 / 9折 / 85折 / 自定义促销
3. 自动识别活动后亏损商品
4. 输出“可参加活动池”与“禁止参加活动池”

### 场景 C：新品定价

1. 录入成本与平台费率
2. 选择目标净利润或目标净利率
3. 系统反推建议售价
4. 对比竞品价格与价格指数
5. 若目标价格不现实，提示需要降成本或降低利润目标

### 场景 D：老板看经营

1. 打开首页看净利润、广告效率、库存风险
2. 查看利润 TOP SKU / 亏损 SKU / 高销量低净利 SKU
3. 看本周待处理策略与风险商品池
